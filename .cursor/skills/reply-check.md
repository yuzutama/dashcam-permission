# reply-check

送信済みリプライへの返信確認を行うスキル。

## トリガー

- 「返信チェックして」
- 「`npm run check` して」
- 「許可 / 拒否 / 返信あり / 判定不可 の意味を教えて」

## 手順

1. プロジェクトルートにいることを確認する
2. `npm run check` を実行する
3. 次を短く報告する
   - 何件チェックしたか
   - 返信が見つかったアカウント
   - 各判定結果
   - 自動解析が走ったかどうか

## 管理画面での見え方

- `許可`: `許可済み` タブへ進む
- `拒否`: `拒否` タブへ進む
- `返信あり`: `送信済み` に残る
- `判定不可`: `送信済み` に残る

## 注意

- `npm run check` の結果を正として扱う
- 返信がなかった場合は、そのことを明確に伝える
- エラー時は推測せず、失敗箇所だけを簡潔に伝える
